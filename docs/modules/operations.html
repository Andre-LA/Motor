Module operations

operations

Table with simple operations that are not compatible between versions 5.1 and 5.3 this is done by `require ('motor.operations.' .. _VERSION:gsub('([%s%.])', ''):lower() .. '_operations')`

Usage:

    local operations = require "motor.operations"

lshift (left, right)
the << operation.

Parameters:

  • left number
  • right number

Returns:

    number result

Usage:

    local operations = require 'motor.operations'
local lshift = operations.lshift
local a = 1 -- 0b0001
print(lshift(a. 1)) -- prints "2" (0b0010)
print(lshift(a. 2)) -- prints "4" (0b0100)
print(lshift(a. 3)) -- prints "8" (0b1000)

combine_2_array_tables (t1, t2)
combines two arrays in one.

Parameters:

  • t1 table first array
  • t2 table second array

Returns:

    table t1 and t2 merged

Usage:

    local operations = require 'motor.operations'
local a = {0, 1, 2}
local b = {3, 4, 5}
local r = operations.combine_2_array_tables(a, b) -- {0, 1, 2, 3, 4, 5}
