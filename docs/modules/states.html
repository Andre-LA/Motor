add_state_data (state, state_data_content)
Adds a new StateData in the State

Parameters:

  • state state the state to add a new state data
  • state_data_content state data content (what this state data actually is), state data can be any type of value

Returns:

    bit_ids.bit_id bit_id of the new state data

See also:

    bit_ids.bit_id
new_systems_step (state, systems_step)
Creates a new system step, a system step is a group of systems , the next group only runs after the current group finishes , for now, this is the only way to run systems in a sequential order

Parameters:

  • state state state to add systems step
  • systems_step {systems.system} array of systems to run

See also:

    systems.system
run (state)
